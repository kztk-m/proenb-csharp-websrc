# プログラミング演習B (C#) 講義資料サイト

このリポジトリは、プログラミング演習B (C#) の講義資料サイトのソースコードを含んでいます。
静的サイトジェネレータ [Hakyll](https://jaspervdj.be/hakyll/) を使用して構築されています。

## コマンドの利用方法

### `site.sh` (サイトの生成・ローカル確認)

`site.sh` は Hakyll によるサイト生成コマンドをラップしたスクリプトです。主な用途は以下の通りです。

- **サイトのビルド**
  ```bash
  sh ./site.sh build
  ```
  `websrc/` 内のファイルを元に `_site/` ディレクトリに静的ファイルを生成します。

- **ローカルでのプレビュー (推奨)**
  ```bash
  sh ./site.sh watch
  ```
  ローカルサーバー（デフォルトで `http://localhost:8000`）を起動し、ファイルの変更を監視して自動的に再ビルドします。執筆中の確認に最も便利です。

- **クリーンビルド**
  ```bash
  sh ./site.sh rebuild
  ```
  キャッシュ（`_cache/`）を破棄してからビルドし直します。不整合が疑われる場合や、最終的なデプロイ前などに実行します。

- **生成ファイルの削除**
  ```bash
  sh ./site.sh clean
  ```
  `_site/` および `_cache/` ディレクトリを削除します。

### `deploy.sh` (サイトのデプロイ)

`deploy.sh` はビルドしたサイトを GitHub Pages (main ブランチ) へデプロイするためのスクリプトです。

**実行条件:**
- 作業ツリーがクリーンであること（未コミットの変更がないこと）。
- 現在のブランチが最新の状態（push済み）であること。

**実行内容:**
1. `sh ./site.sh rebuild` を実行し、サイトを完全に再生成します。
2. リポジトリを一時ディレクトリ（`./deploy`）にクローンします。
3. `_site/` の内容を一時ディレクトリにコピーします。
4. 一時ディレクトリから `main` ブランチへコミット・プッシュします。

```bash
sh ./deploy.sh
```

## ディレクトリ構成

- `websrc/`: サイトのソースファイル（Markdown, CSS, 画像など）が格納されています。
- `gen/`: Hakyll のサイト定義（`site.hs`）やビルド設定（`.cabal`）が含まれています。
- `_site/`: ビルドされた静的ファイルが出力される場所です（Git管理対象外）。
- `_cache/`: Hakyll のビルドキャッシュです（Git管理対象外）。
